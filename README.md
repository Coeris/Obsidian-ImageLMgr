# ImageLMgr - 网图管理

Obsidian 图片链接管理插件，支持全库图片扫描、本地图片管理、图床上传与比对。

## 功能

### 核心功能
- **全库图片扫描**：自动扫描库中所有 Markdown 文件的图片链接（支持 Markdown `![]()` 和 Wiki `![[ ]]` 两种格式）
- **图片分类**：自动识别本地图片、HTTPS 图片、HTTP 图片
- **使用追踪**：统计每张图片在库中的使用次数和来源笔记
- **双击跳转**：双击图片条目可直接跳转到对应的来源笔记
- **图床上传**：支持 GitHub、阿里云 OSS、腾讯云 COS、SM.MS 四种图床
- **本地-云端比对**：检查本地图片是否已上传至图床
- **批量上传**：一键上传所有未上传的本地图片
- **图床文件管理**：查看、插入、删除云端文件，支持目录树展示

### 高级功能

#### #5 图片去重（Hash 检测）
- 基于 SHA-256 文件哈希自动检测重复图片，避免重复上传
- LRU 缓存策略（最多 2000 条），缓存持久化到本地存储
- 上传时自动匹配已有记录，命中则直接复用 URL
- 批量上传时分别报告「新上传」「去重命中」「失败」数量

#### #6 图床连接健康检测
- 工具栏实时显示当前图床连接状态（已连接/连接失败/未检测）
- 点击状态指示器可手动触发检测
- 打开视图/刷新时自动执行连接测试
- 各图床采用针对性的轻量级检测方式：
  - GitHub → 请求仓库信息 API（细分 401/403/404 错误）
  - 阿里云 OSS → ListObjects V2（max-keys=1，空桶视为成功）
  - 腾讯云 COS → ListObjects（max-keys=1）
  - SM.MS → 请求上传历史接口验证 Token
- 检测中显示脉冲动画提示

#### #10 Frontmatter 级别控制
- 支持在笔记头部的 YAML 区域配置该文件的图片管理行为
- 可用字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `image-bed` / `imageBed` | 字符串 | 覆盖全局默认图床（如 `GitHub`、`aliyun`） |
| `auto-upload` / `autoUpload` | 布尔 | 是否允许自动上传（设为 `false` 禁止上传） |
| `image-path` / `imagePath` | 字符串 | 云端路径前缀（如 `blog/2026/`） |

- 使用示例：
  ```yaml
  ---
  image-bed: aliyun
  auto-upload: true
  image-path: blog/
  ---
  ```
- 设置面板内置 Frontmatter 使用说明指南

#### #11 WebDAV 同步增强（冲突检测 + 自动同步）
- **自动同步开关**：开启后每次保存设置自动上传到 WebDAV 服务器
- **智能下载合并**：从服务器下载时自动检测时间戳冲突
- **冲突检测机制**：
  - 通过 `_syncedAt` 时间戳比较本地与远程配置更新时间
  - 差异超过 5 秒视为潜在冲突
  - 冲突时弹窗提示用户选择覆盖方向（远程覆盖本地 / 本地覆盖远程）
  - 无冲突时自动静默合并
- 同步操作按钮：手动上传配置 / 从服务器下载（智能合并）

## 使用

1. 点击左侧功能区图片图标，或使用命令面板执行「打开图床管理面板」
2. 插件会在新标签页中打开，展示全库所有图片链接
3. 双击任意图片条目可跳转到对应的笔记
4. 工具栏显示图床连接状态，可点击手动检测
5. 对未上传的本地图片点击「上传」（自动跳过重复图片）
6. 使用「批量上传」一键处理所有未上传图片

## 设置

在 Obsidian 设置 → 第三方插件 → ImageLMgr 中配置图床信息。

### 支持的图床

| 图床 | 需要配置 |
|------|----------|
| GitHub | Token、Owner、Repo、Branch、Path |
| 阿里云 OSS | Endpoint、Bucket、AccessKey ID、AccessKey Secret |
| 腾讯云 COS | SecretId、SecretKey、Bucket、Region |
| SM.MS | API Token |

### WebDAV 多设备同步

1. 准备一个支持 WebDAV 的服务（如坚果云、NextCloud）
2. 在设置中启用 WebDAV 并填写服务器地址、用户名、密码
3. 建议开启「自动上传配置」，实现多设备无缝同步
4. 点击「从服务器下载（智能合并）」时会自动检测并提示冲突

## 支持的图片语法

- `![](image.png)` - Markdown 标准格式
- `![](image.png|500)` - 带尺寸参数
- `![[image.png]]` - Obsidian Wiki 格式
- `![[image.png|500]]` - Wiki 格式带尺寸
- `![](https://example.com/img.png)` - 网络图片

## 命令

- `打开图床管理面板` - 打开插件主界面
- `刷新图片扫描` - 重新扫描全库图片
